<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| project row → style asset row | Project `styleAssetId` references a style asset that may later be owned by the same user or be a system style. |
| style asset → media object | Style preview crosses from style metadata to media storage; only media IDs are allowed at this layer. |
| schema → generated Prisma client | Generated types become the contract consumed by resolver and later API phases. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | I | `NovelPromotionProject.styleAssetId` | mitigate | Add only a nullable relation in schema; access filtering is implemented in resolver/API plans by querying accessible rows and returning generic fallback. |
| T-01-02 | I | `GlobalStyle.previewMediaId` | mitigate | Store `previewMediaId` with `MediaObject` relation `GlobalStylePreviewMedia`; prohibit preview URL/storage key/signed URL fields in schema parity test. |
| T-01-03 | T | `GlobalStyle.positivePrompt` / `negativePrompt` | mitigate | Keep positive and negative prompt as separate DB columns so later prompt assemblers cannot accidentally merge negative prompt into non-image text prompts. |
| T-01-04 | R/I | Prisma generated contract | mitigate | `GlobalStyle` is additive and db push is blocking; destructive prompts are a hard stop, preserving old project style fields. |
</threat_model>
