<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| task payload → resolver | A serialized snapshot may bypass live style lookup and must be validated before use. |
| project/user input → GlobalStyle lookup | `styleAssetId` may reference another user's private style and must not disclose existence. |
| resolver → prompt consumers | Positive and negative prompt text cross into later generation code and must remain separated. |
| resolver → task retry | Snapshot captures source identity and prompt text to prevent drift after edits. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | I | `resolveStyleContext` style asset lookup | mitigate | Query `globalStyle.findFirst` with `id` plus accessible owner/system visibility; missing and inaccessible assets both return `style-asset-missing-or-inaccessible`. |
| T-01-02 | I | system style projection and resolver output | mitigate | Resolver exposes no raw storage keys or signed URLs; style preview media remains represented by schema `previewMediaId` only. |
| T-01-03 | T | resolver prompt contract | mitigate | `ResolvedStyleContext` and `StylePromptSnapshot` keep `positivePrompt` and `negativePrompt` as separate fields; tests assert separation. |
| T-01-04 | R/I | `StylePromptSnapshot` | mitigate | Snapshot is versioned, serializable, contains identity/source/fallback metadata, and copies prompt strings at capture time. |
</threat_model>
