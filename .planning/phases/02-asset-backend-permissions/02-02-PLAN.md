<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> `/api/assets` style body | User-controlled style prompts, tags, folder and preview media IDs enter the write path. |
| route auth -> `GlobalStyle` write service | Authenticated user identity must be applied to every style write. |
| system style id -> write route | Runtime system assets are visible in reads but must remain non-writable. |
| React Query cache -> asset hub UI | Style cache invalidation must not stale existing asset center state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | S/E | style CRUD routes | mitigate | Global style CRUD uses `requireUserAuth()` and service filters by `userId`. Project-scoped style writes are rejected. |
| T-02-02 | I | preview media body/response | mitigate | Write path accepts only `previewMediaId`; read path from Plan 01 strips `storageKey` and signed URL fields. |
| T-02-03 | T | system styles | mitigate | Update/delete helpers filter `source: 'user'`; system runtime IDs and DB system rows are not writable by users. |
| T-02-04 | D | React Query invalidation | mitigate | Use the existing `invalidateScopeQueries()` global asset invalidation path instead of custom cache keys. |
</threat_model>
