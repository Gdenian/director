<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `GlobalStyle` DB row -> asset API response | Style prompt and media metadata cross from database to browser-facing JSON. |
| media object -> style preview `MediaRef` | Internal media rows contain storage information that must not be exposed in style summaries. |
| legacy system styles -> unified assets | Runtime system styles become visible assets but remain read-only. |
| `AssetKind` union -> existing UI/data consumers | Adding `style` must not break existing character/location/prop/voice narrowing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | I | `readAssets` user scoping | mitigate | Query user style rows only with `userId`; runtime system styles are public read-only projections. |
| T-02-02 | I | `StyleAssetSummary.previewMedia` | mitigate | Build a public media object by explicit field copy and test `storageKey` absence. |
| T-02-03 | T | system style projection | mitigate | Map system styles with `readOnly: true`, no DB write path in this plan. |
| T-02-04 | D | `AssetKind` expansion | mitigate | Update registry/filter/query type tests so existing asset kinds remain present and ordered. |
</threat_model>
