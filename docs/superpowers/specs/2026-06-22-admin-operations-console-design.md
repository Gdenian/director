# 管理者运营控制台设计

## 背景

`director` 当前用户端已经覆盖首页创建作品、工作区创作流程、资产中心、个人设置、创作引擎、模型选择和扣费记录。平台上线前缺少一个只面向管理者的运营控制台，用于管理网站运行规则、用户访问、额度、计费、模型可用性、任务状态和系统健康。

这个控制台不是用户内容管理后台。管理员不负责查看、编辑或控制用户的创作内容。后台设计的第一性原理是回答这些问题：

- 谁可以使用网站？
- 不同用户可以使用哪些能力？
- 用户可以使用多少资源？
- 费用、余额、扣费和充值是否正确？
- 模型、队列、worker 和基础设施是否健康？
- 出现系统性问题时，管理员如何关闭入口、恢复任务或通知用户？
- 管理员的操作是否可追溯？

## 目标

- 普通用户登录后看不到运营控制台入口。
- 普通用户即使手动访问后台页面或后台 API，也不能进入或调用。
- 管理员登录后可以进入运营控制台，处理平台运营事务。
- 站点所有者可以管理管理员、高危财务配置和系统级配置。
- 第一版后台聚焦平台控制面，不做用户创作内容管理。

## 非目标

- 不查看或编辑用户小说正文、分镜内容、提示词、生成图片、生成视频或生成音频。
- 不替用户修改作品、剧集、角色、场景、道具或分镜。
- 不做内容审核工作台，除非后续合规需求单独立项。
- 不做复杂 RBAC 权限矩阵，第一版采用 `user / admin / owner` 三角色。
- 不做独立 CMS 或营销落地页管理系统。

## 角色模型

系统增加三种用户角色：

```text
user
普通用户。只能访问用户端功能。

admin
运营管理员。可以进入运营控制台，处理日常运营、用户状态、额度、任务、模型状态、公告等事务。

owner
站点所有者。拥有 admin 的全部能力，并且可以管理管理员账号、系统级配置、高危财务操作和审计日志。
```

长期数据模型应在 `User` 上持久化角色：

```text
User.role = "user" | "admin" | "owner"
```

第一位 owner 可以通过环境变量或初始化脚本指定，随后由 owner 在后台提升其他用户为 admin。

## 访问控制

### 导航入口

普通用户登录后，导航仍保持用户端入口：

```text
首页
工作区
资产中心
个人设置
```

当 `session.user.role` 为 `admin` 或 `owner` 时，导航额外显示：

```text
运营控制台
```

### 页面保护

后台页面使用本地化路由：

```text
/[locale]/admin
```

访问规则：

- 未登录用户访问后台页面：跳转登录页。
- `role = user` 访问后台页面：返回 403 或跳回首页。
- `role = admin | owner`：允许访问。

### API 保护

后台 API 统一放在：

```text
/api/admin/**
```

所有后台 API 必须在服务端校验 session 和 role。不能只依赖前端隐藏按钮。

访问规则：

- 未登录：401。
- 已登录但不是 `admin` 或 `owner`：403。
- `owner` 专属接口需要额外校验 `role = owner`。

## 后台信息架构

第一版运营控制台包含以下页面：

```text
运营总览
用户管理
用户组与权限
计费与余额
套餐与兑换码
模型与渠道
任务监控
系统健康
公告与功能开关
管理员审计
```

### 运营总览

目的：让管理员一眼判断网站是否正常运行。

展示指标：

- 今日新增用户
- 今日活跃用户
- 今日任务量
- 今日充值金额
- 今日消耗金额
- 失败任务数
- 排队任务数
- 模型异常数
- 系统健康状态

运行状态：

- 文本、图片、视频、语音任务成功率
- text、image、video、voice 队列长度
- 近 24 小时失败任务趋势
- 近 7 天收入与模型成本趋势
- 当前异常列表：卡死任务、扣费异常、模型不可用、支付回调失败

### 用户管理

目的：管理账号状态和用户运营属性，不管理用户创作内容。

用户列表字段：

- 用户名
- 邮箱
- 注册时间
- 最近登录
- 账号状态
- 角色
- 用户组
- 余额
- 冻结金额
- 累计消费
- 任务数量

用户详情展示：

- 基本资料
- 余额与交易流水
- 最近任务元数据
- 失败任务元数据
- 模型配置摘要
- 并发限制
- 登录记录
- 客服备注

允许操作：

- 禁用账号
- 恢复账号
- 调整用户组
- 调整并发限制
- 查看余额和交易流水
- 清空异常冻结金额
- 踢出登录
- 添加客服备注

限制：

- 不展示用户创作正文、图片、视频或音频内容。
- 不提供替用户编辑作品的能力。

### 用户组与权限

目的：用分组方式控制不同用户能用哪些功能和资源。

默认用户组：

- 普通用户
- 测试用户
- VIP 用户
- 内部用户
- 受限用户
- 封禁用户

用户组配置：

- 注册送余额
- 每日任务上限
- 并发上限
- 可用模型范围
- 是否允许视频生成
- 是否允许语音生成
- 是否允许高级模型
- 最大单次任务成本
- 最大未完成任务数

### 计费与余额

目的：保证余额、冻结、扣费、充值、退款可查且可纠正。

页面分区：

- 余额总览
- 交易流水
- 冻结记录
- 使用成本
- 充值订单
- 退款记录
- 异常账单

异常账单类型：

- 支付成功但余额未到账
- 任务失败但扣费
- 任务成功但未扣费
- 冻结金额长期未释放
- 重复扣费
- 余额为负

允许操作：

- 人工充值
- 人工扣款
- 退款
- 解冻余额
- 补单
- 重新查询支付状态
- 导出账单

高危规则：

- 第一版所有人工加款、扣款、退款和支付配置修改仅 owner 可执行。
- 后续如需要让 admin 处理小额纠错，必须先增加可配置金额上限和二次确认流程。
- 所有财务操作必须填写原因并写入审计日志。

### 套餐与兑换码

目的：支持商业化和运营活动。

管理对象：

- 充值套餐
- 会员套餐
- 注册送额度
- 活动赠额
- 兑换码
- 优惠码

兑换码配置：

- 固定金额
- 指定用户组
- 有效期
- 使用次数
- 批量生成
- 是否可叠加

套餐配置：

- 价格
- 赠送额度
- 有效期
- 每日生成上限
- 视频生成次数
- 高级模型权限

### 模型与渠道

目的：管理 AI 模型供应链和平台可用能力。

管理对象：

- OpenAI-compatible 服务
- Gemini
- FAL
- 火山
- 阿里百炼
- 自定义 Provider
- 图片模型
- 视频模型
- 语音模型
- 文本模型
- 口型同步模型

模型表字段：

- 供应商
- 模型名
- 用途
- 状态
- 成功率
- 失败率
- 平均耗时
- 今日调用量
- 今日成本
- 用户可见状态

允许操作：

- 测试连接
- 启用或禁用模型
- 标记维护中
- 调整模型价格
- 调整可用用户组
- 设置推荐模型
- 查看最近失败任务元数据

### 任务监控

目的：观察和恢复平台任务系统，不检查用户内容。

任务列表字段：

- 任务 ID
- 用户
- 任务类型
- 状态
- 进度
- 排队时间
- 运行时间
- 失败原因
- 扣费状态
- worker 类型

状态筛选：

- 排队中
- 运行中
- 成功
- 失败
- 卡死
- 已取消
- 待重试
- 扣费异常

允许操作：

- 重试任务
- 取消任务
- 清理卡死任务
- 重新入队
- 查看错误日志

限制：

- 不展示任务 payload 中的用户正文、提示词或媒体内容。
- 错误日志需要脱敏 API Key、密钥、用户输入正文和媒体 URL。

### 系统健康

目的：让管理员知道基础设施是否正常。

展示内容：

- MySQL 状态
- Redis 状态
- MinIO / S3 状态
- BullMQ 队列状态
- worker 在线状态
- 错误日志
- API 平均耗时
- 最近部署时间
- 环境配置检查
- 日志下载入口

### 公告与功能开关

目的：管理员无需发版即可控制网站开放状态。

公告类型：

- 站内公告
- 维护公告
- 模型故障提示
- 首页公告

功能开关：

- 注册开关
- 充值开关
- 视频生成开关
- 语音生成开关
- 高级模型开关
- 新功能灰度开关
- 指定用户组可见开关

### 管理员审计

目的：所有后台操作都可追溯。

管理员角色：

- admin
- owner

审计日志字段：

- 管理员 ID
- 管理员角色
- 操作类型
- 操作对象
- 操作前数据
- 操作后数据
- 操作原因
- IP
- User-Agent
- 操作时间

必须审计的操作：

- 修改余额
- 禁用或恢复用户
- 调整用户组
- 关闭模型
- 修改套餐
- 修改支付配置
- 新增管理员
- 移除管理员
- 修改管理员角色

## 权限边界

### admin 可以执行

- 查看运营总览
- 查看用户列表和用户运营信息
- 禁用或恢复普通用户
- 调整普通用户的用户组
- 调整普通用户并发限制
- 查看任务状态和错误元数据
- 重试或取消任务
- 查看计费流水
- 管理公告
- 查看模型健康
- 临时关闭模型

### owner 专属

- 创建管理员
- 移除管理员
- 修改管理员角色
- 人工加款、扣款或退款
- 修改系统级计费规则
- 修改支付配置
- 查看完整审计日志
- 清理系统日志

## 数据与隐私边界

后台默认只展示运营元数据：

- 用户账号信息
- 余额和交易
- 任务状态
- 模型状态
- 队列状态
- 系统健康
- 脱敏错误信息

后台默认不展示用户创作数据：

- 小说正文
- prompt
- 角色设定正文
- 分镜描述
- 用户上传原图
- 生成图片
- 生成视频
- 生成音频

如果未来需要合规审核，应单独设计内容审核权限、留痕、告知和数据访问范围。

## 实施分期

### 第一阶段：上线必需

- `user / admin / owner` 角色模型
- 导航栏按角色显示运营控制台入口
- 后台页面和后台 API 服务端鉴权
- 运营总览
- 用户管理
- 计费与余额
- 模型与渠道
- 任务监控
- 系统健康
- 基础审计日志

### 第二阶段：运营增强

- 用户组与权限
- 套餐与兑换码
- 公告与功能开关
- 更完整的财务异常处理
- 管理员账号管理

### 第三阶段：规模化运营

- 更细角色权限
- 风控规则
- 自动化告警
- 成本与收入分析
- 外部支付或客服系统集成

## 验收标准

- 普通用户登录后看不到运营控制台入口。
- 普通用户手动访问 `/:locale/admin` 会被拒绝。
- 普通用户调用 `/api/admin/**` 返回 403。
- `admin` 和 `owner` 可以进入运营控制台。
- `admin` 无法执行 owner 专属操作。
- 后台页面不展示用户创作内容。
- 所有高危操作写入审计日志。
- 任务监控和模型监控只展示必要元数据和脱敏错误信息。
